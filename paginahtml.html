HTML

Significa lenguaje de marcado de diseño, se utiliza para definir la estructura y el diseño de una pagina web, cómo se ve una pagina y cualquier función especial, utilizando etiquetas que tienen atributos.

Linea de tiempo de cambios a HTML5

HTML ha estado en continua evolución desde que se introdujo en Internet a principios de los años noventa. Algunas características fueron introducidas en las especificaciones; otros fueron introducidos en versiones de software. En algunos aspectos, las implementaciones y las prácticas de desarrollo web han convergido entre sí y con las especificaciones y estándares, pero de otras maneras, han divergido.

HTML4 se convirtió en una recomendación de W3C en 1997. Si bien sigue siendo una guía aproximada de muchas de las características principales de HTML, no proporciona suficiente información para crear implementaciones que interactúen entre sí y, lo que es más importante, con contenido web. Lo mismo ocurre con XHTML1, que define una serialización XML para HTML4, y DOM Level 2 HTML, que define las API de JavaScript para HTML y XHTML. HTML reemplaza estos documentos. [DOM2HTML] [HTML4] [XHTML1]

La especificación HTML refleja un esfuerzo, iniciado en 2004, para estudiar las implementaciones contemporáneas de HTML y el contenido web. La especificación:

  1. Define un solo lenguaje llamado HTML que se puede escribir en sintaxis HTML y en sintaxis XML.
  1. Define modelos de procesamiento detallados para fomentar implementaciones interoperables.
  1. Mejora el marcado de los documentos.
  1. Presenta el marcado y las API para los idiomas emergentes, como las aplicaciones web.

¿Qué es html5?

Es la quinta versión del formato utilizado para crear páginas web y aplicaciones, contiene potentes capacidades para aplicaciones basadas en web con mas interactivo, soporte de vídeo, gráficos, más efectos de estilo y un conjunto completo de API.

Elementos estructurales

Elementos estructurales HTML5 con descripciones

Elemento HTML5 Descripción
<header> Introducción de "elementos de sección": un artículo, una sección, todo el documento (página de encabezado). Normalmente, el encabezado de un sitio web que aparece en la parte superior de cada página, o el encabezado de un largo <article> o de un largo <section>
<footer> Contiene el pie de página de un sitio, un largo <article> o un largo <section>
<nav> Sección que contiene los principales enlaces de navegación (dentro del documento u otras páginas).
<article> Contenido independiente, que puede extraerse individualmente del documento y ser sindicado (RSS o equivalente) sin penalizar su comprensión. Normalmente una entrada de blog.
<section> Sección genérica utilizada para agrupar diferentes artículos para diferentes propósitos o temas, o para definir las diferentes secciones de un solo artículo. Generalmente se utiliza con un encabezado.
<time> Se utiliza para marcar tiempos y fechas.
<aside> Sección cuyo contenido no está necesariamente relacionado directamente con el contenido principal que lo rodea, pero puede proporcionar información adicional.
<figure>and<figcaption> Se utiliza para encapsular una figura como un solo elemento, y contiene un título para la figura, respectivamente.
<main> El elemento principal representa el contenido principal del cuerpo de un documento o aplicación. El área de contenido principal consiste en contenido que está directamente relacionado o se expande sobre el tema central de un documento o la funcionalidad central de una aplicación. Solo puede haber un elemento <main> en un documento.

Practica 1:

Siempre agregue un encabezado al contenido de sección explícito

Es mejor por razones de accesibilidad, incluir un encabezado (a <h1>, <h2> … <h6>) en cada elemento de sección (<section>, <article>, <nav>, <aside>) y recordar despues llamar al elemento raíz 

                                        <!DOCTYPE html>
                        <html lang="en">
                        <head>
                        <meta charset = "utf8">
                        <title>Blog practica </title>
                        </head>
                        <body>
                        <header>
                                <h1>Simple <span>HTML5</span> blog</h1>
                        </header>
                        </body>
                        </html>

Resultado:

  1. El elemento <article> se diseñó para partes independientes de un documento que eventualmente podría ser sindicado en flujos RSS.

    Un flujo RSS es un contenido dinámico a disposición de los internautas que se actualiza en tiempo real. Incluye titulares, resúmenes y enlaces hacia páginas que desarrollan con más detalle el contenido seleccionado. Son los enlaces que llevan a su sitio, destinados a generar un tráfico de calidad que usted puede auditar.

  1. Los elementos <section> se utilizan para cortar una parte lógica en subpartes.

Un <article> se puede cortar en diferentes elementos de <section>

Uso de encabezados y nuevos elementos de sección (section, article, aside, nav)

Los elementos <section>, <article>, <nav> y  <aside> se denominan "elementos de sección" . Cortan un documento en secciones que llamamos "secciones".

La especificación HTML5 dice que "cada elemento de sección tiene potencialmente un encabezado y también tiene un esquema asociado".

<h1> ... <h6>  se denominan encabezados , y definen el encabezado de una sección (ya sea que se marque explícitamente utilizando elementos de contenido de la sección, o implícitos en el contenido del encabezado). Esto significa que: 

                                
                                        <!DOCTYPE html>
                                        <html lang="es">
                                        <head>
                                        <meta charset = "utf-8">
                                        <title> Practica </title>
                                        </head>
                                        <body>
                                        <h1>Titulo de mi documento</h1>
                                        </body>
                                        </html>

Resultado:

... define el encabezado de una sección implícitamente , mientras que: 

                                
                                        <!DOCTYPE html>
                        <html lang="es">
                        <head>
                        <meta charset = "utf-8">
                        <title> Practica </title>
                        </head>
                        <body>
                        <section>
                                <h1>Titulo de mi documento</h1>
                        </section>
                        </body>
                        </html>

Resultado:

... define el encabezado de la sección explícita (su elemento principal <section> ).

Use múltiples encabezados de diferente rango con el contenido de la sección

Los encabezados posteriores de rango igual o superior inician secciones nuevas (implícitas), los encabezados de encabezados inferiores de rango inferior implican subsecciones que son parte de la anterior. En ambos casos, el elemento representa el encabezado de la sección implícita.

Aclaremos esto mirando un código de ejemplo: 

                                
                                        <!DOCTYPE html>
                        <html lang="es">
                        <head>
                        <meta charset = "utf-8">
                        <title> Practica </title>
                        </head>
                        <body>
                                <sección>
                                        <h1> Este H1 es el encabezado de una sección explícita </h1>
                                        ...
                                        <h2> Este H2 es un subtítulo, parte de la misma sección
                                        (rango inferior) </h2>
                                                ....
                                        <h1> Este H1 inicia una nueva sección implícita en el explícito
                                        sección (rango igual o superior) </h1>
                                        ...
                                        <h2> Este es un encabezado H2 en la nueva sección que tiene
                                                acaba de empezar </h2>
                                                ...
                                </section>
                        </body>
                        </html>

Variables

Es un lenguaje débilmente tipado, lo que significa que solo hay una forma de declarar una variable. Debido a esto ellas son case sensitive, lo que quiere decir que no es lo mismo 'comida' que 'Comida' o 'COMIDA'. Los nombres reservados no pueden ser usados como variables, los estudiaremos en todo el blog. Solo pueden iniciar con los siguientes caracteres: "$", "_", "a" to "z", or "A" to "Z". Es preferible utilizar la notación camelCase, donde la primera letra debe de ser minúscula, y cuando haya un cambio entre palabras, por ejemplo camer a case, la segunda palabra iniciará con mayúscula, y así sucesivamente en caso de que haya más de una palabra. 

                
    //forma 1
    var x;
    var myNumber1, myNumber2 = 34, myNumber3;
    //forma 2
    let x;
    let x = 1, y = 3, z = 12;

    const MAX_GRADE = 20;

What is the difference between == and === in JavaScript?

Equal (==)

Returns true if the operands are strictly equal with type conversion.

Strict equal (===)

Returns true if the operands are strictly equal with no type conversion.

The triple-equals operator never does type coercion. It returns true if both operands reference the same object, or in the case of value types, have the same value.

1 == 1 ;
//true
 
1 == 2 ;
//false
 
/* Here, the interpreter will try to convert the string ‘1’
into a number before doing the comparison */
 
1 == '1';
//true :
 
 
//with strict equal, no conversion:
 
1 === 1;
//true
1 === '1';
//false

Depending on the context, generally strict equal (or strict not equal) is preferred.

Best practice for beginners: always use === or !== for comparisons.

do while

var i = 0;
 
do {
    console.log('i = ' + i);
    i++;
} while(i < 20);
 
console.log('Value of i after the do-while statement: ' + i);

The do-while statement executes the content of the loop once before checking the condition of the while, whereas a while statement will check the condition first before executing the content.

for in

The for-in statement is used to iterate through an object (or through an array, which is also an object).

for ( variable in expression ) statement

var michel = {              // michel is an object
    familyName:'Buffa',     // familyName, givenName, age
                            // are its properties
    givenName: 'Michel',
    age: 51
}
 
for(var property in michel) {   // the for-in will
                                // enumerate properties
    console.log(property);      // will print "familyName",
                                // "givenName",
                                // "age"
    console.log(michel[property]);  // michel['givenName'] same 
                                    // as michel.givenName
}

Eventos

Sin embargo, realizar una acción específica cuando se hace clic en el botón, saber qué botón del mouse se ha utilizado, calcular la posición del puntero del mouse (x, y) en la coordenada del sistema del botón o ejecutar tareas más complejas solo se puede realizar a través de JavaScript.

Con JavaScript, un clic de botón, un movimiento del mouse, una ventana redimensionada y muchas otras interacciones crean lo que se denominan "eventos". El tiempo y el orden de los eventos no se pueden predecir por adelantado. Decimos que el "procesamiento de eventos" es asíncrono. Los navegadores web detectan eventos a medida que ocurren y pueden pasarlos al código JavaScript. Lo hacen permitiéndole registrar funciones como escuchas de eventos , también llamados controladores o devoluciones de llamada para eventos específicos.

Cada vez que ocurre un evento, el navegador lo coloca en una "cola de eventos".

Luego, el navegador mira una lista de "Oyentes de eventos" y llama a los que corresponden al tipo de evento que "escuchan".

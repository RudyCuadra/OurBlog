Variables

Es un lenguaje débilmente tipado, lo que significa que solo hay una forma de declarar una variable. Debido a esto ellas son case sensitive, lo que quiere decir que no es lo mismo 'comida' que 'Comida' o 'COMIDA'. Los nombres reservados no pueden ser usados como variables, los estudiaremos en todo el blog. Solo pueden iniciar con los siguientes caracteres: "$", "_", "a" to "z", or "A" to "Z". Es preferible utilizar la notación camelCase, donde la primera letra debe de ser minúscula, y cuando haya un cambio entre palabras, por ejemplo camer a case, la segunda palabra iniciará con mayúscula, y así sucesivamente en caso de que haya más de una palabra. 

                
        //forma 1
        var x;
        var myNumber1, myNumber2 = 34, myNumber3;
        //forma 2
        let x;
        let x = 1, y = 3, z = 12;
Constantes

Se definen escribiendo el identificador todo en MAYUSCULAS o usando la palabra reservada *const* 

                
        //forma 1
        var TIME_LIMIT;
        //forma 2
        const MAX_GRADE = 20;

        }
        }

Operadores lógicos

Valores booleanos, son verdadero o falso, true or false.
Valor indefinido y null, retorna undefined cuando una varable no ha sido asignada

  • &&(AND)
    • if ((x > 0) && (x < 10)) {  console.log('x tiene que ser extrixtamente positivo y menor que 10');  }
  • ||(OR)
    • if ((x > 0) || (x == -5)) {  console.log('x es positivo o igual a -5');  }
  • !(NOT)
    •   if (!(x > 0)) {  console.log('x no es positivo (x es menor o igual a 0');  }
&&, || son operadores binarios, ! is unitario.
Todos los valores siguientes son evaluados como falso
  • false
  • undefined
  • null
  • 0
  • NaN
  • Las cadenas vacias ""

¿Cuál es la diferencia entre igual y estrictamente igual?

Igual(==)
Retorna igual si el operador es estrictamente igual con el tipipo de conversión
Esctrictamente igual(===)
Retorna verdadero si el operador es estrictamente igual sin conversión de tipo, este operador nunca hace referencia de tipo. Retorna verdadero únicamente si ambos operadores hacen referencia al mismo objeto, o si en el caso de valores de tipo, tienen el mismo valor. 

                        
        1 == 1 ;
        //true
        
        1 == 2 ;
        //false
        
        /* Aqui, el interpretador va a intentar convertir la cadena '1' a un numero antes de hacer la comparacion */
        
        1 == '1';
        //true :
        
        
        //con estrictamente igual no hay conversion:
        
        1 === 1;
        //true
        1 === '1';
        //false
Dependiendo del contexto, generalmente estrictamente igual es preferido. Buenas prácticas para novatos: siempre usar === o !== para comparación.

for in

El for-in es usado para iterar a través de un objeto(o de un arreglo, que también es un objeto). 

                
                var michel = {              // michel is an object
                familyName:'Buffa',     // familyName, givenName, age
                                        // are its properties
                givenName: 'Michel',
                age: 51
                }
                
                for(var property in michel) {   // the for-in will
                                                // enumerate properties
                console.log(property);      // will print "familyName",
                                                // "givenName",
                                                // "age"
                console.log(michel[property]);  // michel['givenName'] same 
                                                // as michel.givenName
                }

Eventos

Sin embargo, realizar una acción específica cuando se hace clic en el botón, saber qué botón del mouse se ha utilizado, calcular la posición del puntero del mouse (x, y) en la coordenada del sistema del botón o ejecutar tareas más complejas solo se puede realizar a través de JavaScript.
Con JavaScript, un clic de botón, un movimiento del mouse, una ventana redimensionada y muchas otras interacciones crean lo que se denominan "eventos". El tiempo y el orden de los eventos no se pueden predecir por adelantado. Decimos que el "procesamiento de eventos" es asíncrono. Los navegadores web detectan eventos a medida que ocurren y pueden pasarlos al código JavaScript. Lo hacen permitiéndole registrar funciones como escuchas de eventos, también llamados controladores o devoluciones de llamada para eventos específicos.
Cada vez que ocurre un evento, el navegador lo coloca en una "cola de eventos".Luego, el navegador mira una lista de "Oyentes de eventos" y llama a los que corresponden al tipo de evento que "escuchan".
